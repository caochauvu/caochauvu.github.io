Rate Heterogeneity

Models with Rate Heterogeneity

There are sites where mutations do arise, but some sites become fixed and some don't. That is to say, different sites will tend to have different evolution rates, especially if they are far apart (think of recombination distance).

\(r_{1}\) \(r_{2}\) ... \(r_{n}\)
A C ... G
A C ... A
G C ... G

Q matrix (above) is multiplied by \(r_{i}\) for each site \(i\).

Gamma (\(\alpha,\alpha\)) distribution shows different rates.

  • Small \(\alpha\) (e.g. \(\alpha<1\)): High level of rate variation.
  • Large \(\alpha\) (e.g. \(\alpha>1\)): Low level of rate variation.

In practice, we can't estimate for all \(r_{n}\),but we can estimate \(\alpha\), it's easier to say what the sites are doing on average. We can do this by binning all of the \(r\) into four bins (though we can increase or decrease this number, four is just the typically used one), this helps with computational efficiency. We can also model proportion of invariant sites (kind of opposite of rate heterogeneity, accounts for sites that can never vary), e.g. F84+I+G (F84 model with invariant sites and gamma rates).

Basic Genetics

Central dogma: Organization of genetic information, \(DNA \rightarrow RNA \rightarrow Protein\).

Sequencing Technologies: The automation and parallelization of individual steps that gives us the sequence of an organism.

Genetic Terminology:

  • Locus(loci)
  • Markers
  • Allele
  • Polymorphic loci
  • Types of mutation
    • Point mutations
    • Transitions
    • Transversions
  • Indels
  • Microsatellites.

Genetic drift and selection: Neutral mutations can fix in population due to drift. Selection changes the probability of fixation, which allows non-neutral mutations to fit more/ less frequently.

Sequence Alignment

Sequence alignment: Align homologous nucleotides or amino acids.

  • Sequence alignment score: \(\sum S(a,b) - \sum W(l)\). First part is summing over all aligned pairs, where \(S\) is elements of the scoring matrix. The second part sum over all gaps, with \(l\) specifying gap length. Gap penalties are as follow:
    • Linear: \(W(l) = g*l\).
    • Affine: \(W(l) = g_{open} + g_{extension} (l-1)\)

Alignment Algorithms:

  • Needleman-Wunsch Algorithm: Global alignment.
  • Smith-Waterson algorithm: Local alignment.

BLAST: Stands for Basic Local Alignment Search Tool. BLAST is use to find a match for our query sequence in a database.

E-value: Expected # of database sequence matches to our query sequence, with a score of \(\geq S^{*}\).

  • We would simulate random sequence that we can run through BLAST.
  • If \(S^{*}\) is a score of a "good" alignment, then numbers (\(n_{1},...n_{1000}\)) should be low (mostly zeros).

Evolutionary Distance

Hamming Distance: A proportion of polymorphic sites.

$$D = \frac{\#\ of\ polymorphic\ sites}{\#\ of\ total\ sites}$$

Jukes-Cantor Distance:

$$\hat{d} = - \frac{3}{4} \ln (1-\frac{4}{3}D)$$

Take notice that as time progress, the expected (calculated above) and observed distance will diverge.

Subsitution models:

  • What is needed for the exam is to be able to understand nested relationships among the models.
  • E.g. If we artificially make \(\pi_{A} = \pi_{G} = \pi_{C} = \pi_{T}\) in HKY model, it becomes K2P model.

Distance-Based Phylogenetic Reconstruction

Least-square Method: Returns tree with branch length that minimizes:

$$\sum_{i<j}(d_{ij}-e_{ij})^{2}$$

Where \(d_{ij}\) is the sequence distance and \(e_{ij}\) is the tree distance.

Minimum Evolution Method: Uses least square to estimate a branch length for each tree, returns a tree with the minimum total (sum of all branch) length.

UPGMA: An approximation (a faster way that returns close enough results) of least-square but with constraint of ultrametric trees (molecular clock).

NJ: An approximation (same as above) of minimum evolution with no constraints on branch length.

Maximum Parsimony: Returns a phylogeny that has the minimum number of needed substitutions to explain the data given.

Problems:

  • Can't attach parameters (it's hard to extend), or use it to estimate parameters.
  • It's statistically inconsistent. The more data you attached, the less accurate the tree will be.
  • Long-branch attraction, resulting from not being able to attach parameters (like branch length). It's statistically inconsistent.

Maximum Likelihood: Returns a combination of both the phylogeny and the parameters that made up the model that estimated the phylogeny. Maximize the probability of observing the given data.

Problems:

  • Computationally costly.
  • Model needs to be correctly specified (and we can never be completely sure of our parameters for the model we give ML).
